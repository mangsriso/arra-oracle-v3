# Menu Autoload — `ORACLE_MENU_DIR`

Frontend menu items live in `src/menu/frontend.ts` by default. If you set the
`ORACLE_MENU_DIR` environment variable, the server will instead read every
`*.ts` and `*.json` file in that directory at startup and build the frontend
menu from the union of their exports.

This lets deployments ship an external `menu.d/` directory — mounted via a
Docker volume, managed by a config package, generated by a CLI, etc. — without
touching the repository source.

## Behaviour

- **Unset** (default): the server uses the bundled `src/menu/frontend.ts`.
- **Set to an existing directory**: every `*.ts` and `*.json` file in the
  directory is loaded (non-recursive). Files are read in sorted filename order.
- **Set to a missing directory**: a warning is logged and the server falls back
  to the bundled defaults.

## File formats

### `.ts` — default-exported array

```ts
import type { MenuItem } from 'arra-oracle-v3/src/routes/menu/model.ts';

const items: MenuItem[] = [
  { path: '/canvas', label: 'Canvas', group: 'tools', order: 80, source: 'page' },
  { path: '/planets', label: 'Planets', group: 'tools', order: 81, source: 'page' },
];

export default items;
```

### `.json` — direct array of `MenuItem`

```json
[
  { "path": "/map", "label": "Map", "group": "tools", "order": 82, "source": "page" },
  { "path": "/settings", "label": "Settings", "group": "hidden", "order": 99, "source": "page" }
]
```

See `src/routes/menu/model.ts` for the full `MenuItem` shape.

## Merge order and duplicates

- Files are loaded in **sorted filename order** (alphabetical).
- The relative position of items within the merged list preserves the input
  order — the first time a `path` is seen decides where it appears in the
  final array.
- If the **same `path`** is exported by more than one file, **later files
  override earlier files** (the value is replaced; the position is kept).

A common convention is to prefix filenames with a numeric weight:

```
10-pages.ts
20-experimental.ts
30-overrides.json
```

Here `30-overrides.json` wins over `10-pages.ts` for any shared `path`.

This rule is **across file types**: a `.json` file will override a `.ts` file
if it sorts later. In the example above, `30-overrides.json` overrides both
`10-pages.ts` and `20-experimental.ts`.

## Example

```bash
mkdir -p /etc/arra-oracle/menu.d
cat > /etc/arra-oracle/menu.d/10-pages.json <<'EOF'
[
  { "path": "/canvas", "label": "Canvas", "group": "tools", "order": 10, "source": "page" },
  { "path": "/map", "label": "Map", "group": "tools", "order": 20, "source": "page" }
]
EOF

ORACLE_MENU_DIR=/etc/arra-oracle/menu.d bun src/server.ts
```
